PhaseOrder
Temporal ordering of phases. Knowledge of the ordering of phases at a site can be stored here. These can be used as constraints in OxCal input files. For example, if four phases are known at a site, such that p1, p2, p3 are all stratographically ordered (p1 is oldest), and p4 is only known to be older than p3 (but the relationship with p1 and p2 is unknown), we can state the following: p1<p2; p2<p3; p4<p3.
Column name Data type Column description
SiteID varchar
PhaseID_before varchar
PhaseID_after varchar
